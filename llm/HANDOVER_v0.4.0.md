## 3. Technical Architecture Overview

### Core Engine (`R/simulation.R`)
- **`run_simulation()`**: The primary orchestrator. It uses a `credit_policy` object (pipeline of stages).
- **`classify_scenarios()`**: Assigns applicants to quadrants:
    - `keep_in`: Approved by both policies.
    - `swap_in`: Gain (Approved by New, Rejected by Current).
    - `swap_out`: Loss (Approved by Current, Rejected by New).
    - `keep_out`: Rejected by both.

### Risk Grouping (`R/grouping.R`)
- **`find_risk_groups()`**: The "brain" for clustering. It uses `max_crossings` to merge clusters that would result in unstable RBP matrices (where lower risk Rating has higher PD than the next one in some months).
